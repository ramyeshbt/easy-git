Getting Started

Installation

Clone the repo, run the installer, and you're done. The tool is added to your PATH and available in every new shell session.

bash
# Clone
git clone https://github.com/ramyeshbt/easy-git.git
cd easy-git

# Install (adds bin/ to PATH in ~/.bashrc or ~/.zshrc)
bash install.sh

# Open a new shell, then in any git repo:
g status
g --doctor    # verify all dependencies
ℹ
Idempotent installer
Running bash install.sh twice is safe — it won't duplicate PATH entries. To remove the tool completely, run bash uninstall.sh.
◎
Optional dependencies
fzf enables fuzzy branch/stash search. gh (GitHub CLI) enables PR creation. Both are optional — the tool gracefully degrades without them.
Design

Architecture

A single entry point routes every subcommand to a dedicated script. All scripts share one core library — no hidden globals, no side effects.

g commit
User input
──▶
bin/g
Entry point
──▶
lib/commit.sh
Subcommand
──▶
lib/core.sh
Shared utils
──▶
git
Git CLI
⬡
Single entry point bin/g dispatches to lib/<cmd>.sh — adding a subcommand is one file + one dispatch line.
◈
Shared core Every lib script sources lib/core.sh for colors, TTY-safe I/O, confirm prompts, and dry-run support.
✦
Graceful degradation fzf absent → numbered list. gh absent → PR URL skipped. The tool always works at a basic level.
◎
Bash 3.2+ compatible macOS ships bash 3.2. No ${var,}, no declare -A, no readarray — verified by CI on every commit.

Security

Security Model

easy-git applies defense-in-depth to prevent accidental data loss and credential leakage.

🛡
Zero eval policy
There is no eval anywhere in the codebase. Dynamic dispatch uses case statements only. Verified by grep -r 'eval ' in CI on every commit.
🔑
Secret detection pre-commit Blocks commits containing AWS keys (AKIA…), GitHub tokens (ghp_…), GitLab tokens, npm tokens, private key headers, DB URIs with credentials, JWT tokens, and sensitive filenames (.env, .pem, *_rsa, etc.)
🌿
Branch name sanitization Input is passed through a whitelist: [a-zA-Z0-9._/-] only. Leading dashes, .lock suffix, and shell metacharacters are stripped before any git command is run.
🛑
Protected branch warnings Pushing to main, master, develop, release, production, or staging triggers a prominent warning and requires explicit confirmation.
⚓
--force-with-lease only No command ever uses git push --force. All force-push paths use --force-with-lease to prevent overwriting others' work on shared branches.
🔒
-- separator enforcement All git commands that accept user-supplied refs use -- to separate options from arguments, preventing flag injection via branch names or filenames.
📺
TTY-safe I/O All interactive reads use a [ -t 0 ] guard with stdin fallback — no crashes in CI, Docker, or VS Code integrated terminals. Verified in the test suite.
Configuration

Environment Variables

Control tool behavior with shell environment variables — no config file needed.

Variable Effect Example
G_DRY_RUN=1 Preview all commands without executing — prints [dry-run] git … G_DRY_RUN=1 g sync
G_NO_COLOR=1 Disable all color output (auto-disabled when stdout is not a TTY) G_NO_COLOR=1 g log
VERBOSE=1 Verbose output in the test runner — prints each individual assertion VERBOSE=1 bash tests/run_tests.sh
◈
Dry-run in CI pipelines
G_DRY_RUN=1 is useful for validating what g sync or g push would do inside a CI job without actually modifying the remote. Combine with VERBOSE=1 for maximum debug output.
Compatibility

Pass-through

Any command that easy-git doesn't recognize is passed directly to git. You never lose access to the full git surface.

bash
# These all work transparently:
g cherry-pick abc123   # → git cherry-pick abc123
g bisect start         # → git bisect start
g rebase -i HEAD~5     # → git rebase -i HEAD~5
g worktree list        # → git worktree list
g submodule update     # → git submodule update
g diff HEAD~3          # → git diff HEAD~3
✦
Zero lock-in
easy-git is purely additive. Every git command you already know keeps working as-is. The tool enhances the most common 20 workflows; everything else passes through unchanged.
